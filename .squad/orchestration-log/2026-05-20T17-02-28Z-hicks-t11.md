## Summary

Shipped 3 new reporting API endpoints for inventory summary, warranty expiry tracking, and historical spending analysis. Added `Device.WarrantyExpiry` field via migration. All endpoints use EF projections for efficiency, FluentValidation for input contracts, and OpenAPI documentation.

## Endpoints Delivered

1. **GET /api/v1/reports/summary**
   - Returns device count by status (Active, Retired, Disposed)
   - Includes total inventory value (sum of PurchasePrice)

2. **GET /api/v1/reports/warranties**
   - Returns devices with WarrantyExpiry, sorted by expiration date
   - Includes device ID, name, brand, warranty expiry date
   - Optional filter: status, expiry threshold

3. **GET /api/v1/reports/spending**
   - Returns historical spending grouped by year/category
   - Treats all devices (Active, Retired, Disposed) with PurchaseDate + PurchasePrice
   - Supports date range filtering

## Architecture Decisions

1. **Persist warranty expiry on Device:** Rationale — reporting spec requires durable warranty-expiry data; existing aggregate lacked this field
2. **Normalize status to three buckets:** Active, Retired, Disposed only; InRepair and Lent roll into Active
3. **Treat spending as historical:** Include retired/disposed devices to preserve purchase history
